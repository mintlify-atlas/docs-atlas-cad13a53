## Endpoint

```
GET /v1/images/{token}.{ext}
```

Serves temporarily stored images that were uploaded via base64 encoding to the `/v1/reason2/action` endpoint. Images are stored in-memory with a time-to-live (TTL) and automatically cleaned up after expiration.

## Path Parameters

<ParamField path="token" type="string" required>
  Unique token identifying the stored image. Generated by the server when using `image_b64` in the reason2-action endpoint.
</ParamField>

<ParamField path="ext" type="string" required>
  File extension matching the image format: `jpg` or `png`. Must match the stored image's extension.
</ParamField>

## Response

Returns the image binary data with the appropriate `Content-Type` header.

<ResponseField name="Content-Type" type="header">
  MIME type of the image:
  - `image/jpeg` for JPEG images
  - `image/png` for PNG images
</ResponseField>

<ResponseField name="Cache-Control" type="header">
  Set to `no-store` to prevent caching.
</ResponseField>

<ResponseField name="Expires" type="header">
  Set to `0` to indicate immediate expiration.
</ResponseField>

## Image Storage

### Time-to-Live (TTL)

Images are stored in-memory for **120 seconds** by default (configurable via `IMAGE_TTL_S` environment variable). After expiration, the image is automatically removed during cleanup cycles.

### Format Detection

The server automatically detects image format using magic bytes:
- JPEG: Starts with `0xFF 0xD8 0xFF`
- PNG: Starts with `0x89 0x50 0x4E 0x47 0x0D 0x0A 0x1A 0x0A`

### Size Limits

Maximum image size is **2MB** by default (configurable via `MAX_IMAGE_BYTES` environment variable).

## Use Case

This endpoint is automatically used when submitting base64-encoded images to the `/v1/reason2/action` endpoint:

1. Client sends `image_b64` to `/v1/reason2/action`
2. Server decodes, validates, and stores the image with a unique token
3. Server constructs a temporary URL: `{PUBLIC_BASE_URL}/v1/images/{token}.{ext}`
4. This URL is passed to the NIM backend for VLM processing
5. Image expires after `IMAGE_TTL_S` seconds

## Examples

<CodeGroup>

```bash curl
# Retrieve a temporary image
curl https://your-api.com/v1/images/AbCdEfGhIjKlMnOpQrStUvWxYz.jpg \
  --output retrieved-image.jpg
```

```python Python
import requests

# Retrieve a temporary image
token = "AbCdEfGhIjKlMnOpQrStUvWxYz"
ext = "jpg"

response = requests.get(
    f"https://your-api.com/v1/images/{token}.{ext}"
)

if response.status_code == 200:
    with open("retrieved-image.jpg", "wb") as f:
        f.write(response.content)
    print("Image retrieved successfully")
else:
    print(f"Error: {response.status_code}")
```

</CodeGroup>

## Error Responses

<ResponseField name="404" type="error">
  Not Found - Image token not found or expired.
  
  Possible causes:
  - Token does not exist in the image store
  - Image has expired (TTL exceeded)
  - Extension mismatch (requested `.jpg` but image is `.png`)
</ResponseField>

## Implementation Details

### In-Memory Storage

Images are stored in a Python dictionary:

```python
# token -> (bytes, mime, ext, expires_at)
_IMAGE_STORE: Dict[str, Tuple[bytes, str, str, float]] = {}
```

### Token Generation

Tokens are generated using `secrets.token_urlsafe(24)`, providing cryptographically secure random URLs.

### Automatic Cleanup

Expired images are removed automatically when:
- A new image is stored via `/v1/reason2/action`
- An image is requested via this endpoint

## Configuration

| Variable | Default | Description |
|----------|---------|-------------|
| `IMAGE_TTL_S` | `120` | Time-to-live for temporary images in seconds |
| `MAX_IMAGE_BYTES` | `2097152` | Maximum image size in bytes (2MB) |
| `PUBLIC_BASE_URL` | `http://127.0.0.1:8080` | Public base URL used to construct image URLs |
